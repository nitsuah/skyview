# Security Policy

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability, please report it by emailing the maintainers directly at **contact@skyviewdynamics.com**.

### What to Include

When reporting a vulnerability, please include:

- Type of issue (e.g., cross-site scripting)
- Steps to reproduce the issue
- Impact of the issue

### Response Timeline

- **Initial Response**: Within 48 hours
- **Updates**: We will keep you updated as we investigate the issue.

## Security Best Practices

When contributing to Skyview:

- **No Secrets**: Never commit API keys, passwords, or tokens (use environment variables).
- **Dependencies**: Keep any included libraries up to date.
